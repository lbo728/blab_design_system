# A11Y.md

## Accessibility Principles
Accessibility is a core quality standard, not an optional enhancement.
BLab components and patterns should remain usable, readable, and navigable by default.

## Contrast Rules
Text and interactive elements should meet measurable contrast requirements.
- Normal text: minimum 4.5:1 contrast ratio
- Large text: minimum 3:1 contrast ratio
- Where practical for key reading surfaces, prefer 7:1 for stronger accessibility support
These thresholds should be testable in design review and implementation review.

## Focus Visibility
Interactive elements must have clear, persistent visible focus states for keyboard users.
- Minimum expectation: a clearly contrastive 2px outline or a distinct 3px focus ring
- Focus indicators must not be removed for visual neatness
- Keyboard-focusable elements should always expose a visible focus treatment
These rules should be verifiable through keyboard navigation testing.

## Touch Target Size
Controls should be comfortably tappable and not rely on precision interaction.
- Minimum touch target: 44x44 CSS pixels (roughly 9mm)
- Adjacent interactive elements should have enough spacing to avoid overlap or accidental taps
These requirements should be checked in component review and UI QA.

## Typography Readability
Typography should favor legibility, spacing, and stable hierarchy over overly expressive styling.

## Icon Accessibility
Icon-only controls must remain understandable through accessible labels and clear affordances.

## Form Accessibility
Inputs should expose labels, helper text, error states, and validation feedback clearly.
Color alone must not carry the entire meaning.

## Dialog and Modal Accessibility
Dialogs should manage focus correctly, preserve hierarchy, and remain dismissible in expected ways.

## Motion Accessibility
Motion should respect reduced motion preferences and avoid causing disorientation.

## Error Message Rules
Errors should be understandable, specific, and supportive.
Do not rely on visual emphasis alone.
